# 要件定義書

---

## 1. システム概要

本システムは、TRPGセッションの日程調整の支援を目的としたWebアプリケーションである。
ユーザーが個人の予定を登録し、セッション単位で参加者の予定を集計することで、最適なセッション日時を決定できるようにする。
本システムはログイン不要で利用可能にし、かつ複数セッション間での予定の再利用性を向上させる。

---

## 2. システム目的

スケジュール調整サービス「伝助」での日程調整で発生した以下問題を解決する

* コミュニティの人口増加に伴い、一つの日程表上に存在する表示が多くなりすぎ、調整役の認知負荷が大きくなっている
* 複数の日程表に予定を書き込む場合、予定の変更が必要な場合にすべての日程表に同じ変更を繰り返し適応する作業が必要になりユーザー負担が大きい
* スケジュールは1日単位かつ数種の記号で表現する制限があり、細かな時間帯や不明瞭な予定については口頭のやり取りが必要で時間を要する

その他

* 小規模コミュニティや複数のコミュニティにまたがるユーザーも想定して安全な予定共有を行う

---

## 3. 想定利用規模

| 項目 | 想定 |
| - | - |
| 総ユーザー数 | 最大100人程度 |
| 総同時セッション数 | 最大100程度 |
| セッションあたり最大ユーザー数 | 20人程度 |
| 予定管理期間 | 最大2年程度 |

---

## 4. 利用者

| 利用者 | 説明 |
| - | - |
| GM | セッションを企画するユーザー |
| PL | セッションに参加するユーザー |

システム上の権限は基本的に同一だが、セッション作成者のみ一部操作権限を持つ。

---

## 5. 機能要件

### 5.1 セッションの作成
ユーザーはセッションを作成できる。セッション作成時にはユーザーは「セッション名」「説明文」「セッションの候補日」を指定し、システムは共有用URLトークンと編集用URLトークンを発行する。

### 5.2 セッションの編集・削除
ユーザーはセッションを編集または削除できる。編集用URLトークンからセッション編集画面にアクセスでき、セッション名や説明文等、セッション候補日を編集したり、セッションを削除することが出来る。

### 5.2 セッションへの参加
ユーザーはセッションへ予定を追加できる。

>「予定」とは、一人のユーザーの各時間帯における 利用可能/不可 を表した情報を指す

ユーザーは参加したいセッションのセッション画面から「新規予定」又は「既存予定」を選び、参加情報を入力することで予定を追加する。

> 参加情報は、表示名(例えばTRPGでの担当キャラクタ名などを設定できる)、メモ(備考や連絡先などを共有できる)、ID公開設定(閲覧ユーザーに編集を許すかの設定:予定の編集にIDが必要)、などであり、(複数セッションで)予定は共有されている場合でも参加情報はセッション毎に独立している。

新規の予定を追加する場合、システムがその予定を一意に表すIDを発行する。既存の予定を追加する場合、IDを入力することで別のセッションに属している予定を追加できる。

### 5.3 予定の編集
ユーザーは予定の編集を行うことができる。ユーザーは、セッション画面から、そこに属している予定を一つ選び、そのIDを入力することで予定編集画面を開ける(IDが公開されている場合、ID入力はスキップされる)。

予定編集画面では、現在の予定情報が表現され、カレンダーのようなGUIから1時間単位で利用可能/不可能/未入力 の3値を更新・保存できる。
同IDの予定はセッション間で共有される(内部的には、同じデータを複数セッションが参照している)ので、複数セッションに参加していても複数回の入力は不要。
また、そのセッションについての参加情報も編集できる。

### 5.3 セッションの閲覧
ユーザーはセッションの共有用URLトークンから、セッション画面を開くことができる。
セッション画面では、参加者全員分の予定が参加可能な時間/不明な時間などを色分けしてカレンダーのように表示できる。また参加者全員の予定を個別に閲覧できる。
