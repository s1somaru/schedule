# 要件定義書

---

## 1. システム概要

本システムは、TRPGセッションの日程調整の支援を目的としたWebアプリケーションである。
ユーザーが個人の予定を登録し、セッション単位で参加者の予定を集計することで、最適なセッション日時を決定できるようにする。
本システムはログイン不要で利用可能にし、かつ複数セッション間での予定の再利用性を向上させる。

---

## 2. システム目的

スケジュール調整サービス「伝助」での日程調整で発生した以下問題を解決する

* コミュニティの人口増加に伴い、一つの日程表上に存在する表示が多くなりすぎ、調整役の認知負荷が大きくなっている
* 複数の日程表に予定を書き込む場合、予定の変更が必要な場合にすべての日程表に同じ変更を繰り返し適応する作業が必要になりユーザー負担が大きい
* スケジュールは1日単位かつ数種の記号で表現する制限があり、細かな時間帯や不明瞭な予定については口頭のやり取りが必要で時間を要する

その他

* 小規模コミュニティや複数のコミュニティにまたがるユーザーも想定して安全な予定共有を行う

---

## 3. 想定利用規模

| 項目 | 想定 |
| - | - |
| 総ユーザー数 | 最大100人程度 |
| 総同時セッション数 | 最大100程度 |
| セッションあたり最大ユーザー数 | 20人程度 |
| 予定管理期間 | 最大2年程度 |
